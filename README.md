# ai school prototype

Минималистичный многостраничный прототип платформы для обучения людей 40+ работе с нейросетями и будущего ИИ-агрегатора.

## Что есть в прототипе

- Рабочее окно с двумя ИИ-панелями: основная задача и правый помощник.
- Раскрывающийся каталог по 5 направлениям: текст, графика, видео, звук, ИИ-агенты.
- Библиотека промптов.
- ИИ-терминология.
- Мой профиль как основа социальной механики.
- Самостоятельное обучение модульной дорожкой.
- Отдельные страницы Telegram бота и MAX бота.
- Настройки с режимами советчика: новичок, базовый пользователь, опытный пользователь.
- Roadmap проекта.
- 100 случайных советов в левом нижнем блоке.
- Светлая/темная тема через круглую кнопку с солнцем/полумесяцем.

## Email-регистрация

Рекомендуемый механизм:

1. Выбрать backend: Supabase Auth, Firebase Auth, Clerk, Auth.js или собственный backend.
2. Подключить SMTP-провайдера: Resend, Postmark, SendGrid, Mailgun или корпоративную почту.
3. Создать таблицу профилей: user_id, email, name, age_group, skill_level, advisor_mode, interests, created_at.
4. Сделать форму регистрации: email, пароль, согласие с правилами и политикой данных.
5. Отправлять письмо подтверждения email.
6. После подтверждения создавать профиль пользователя и открывать анкету "Мой профиль".
7. Добавить восстановление пароля через одноразовую ссылку.
8. Включить защиту: rate limit, проверку одноразовых токенов, запрет слабых паролей, журнал входов.
9. Хранить API-ключи нейросетей только на backend, никогда в браузере.
10. Перед запуском подготовить пользовательское соглашение и политику обработки данных.

## Free-tier API для первого подключения

Эти варианты подходят для MVP, но лимиты нужно проверять перед коммерческим запуском:

- Google Gemini API / ИИ Studio: бесплатный стартовый доступ и free tier для части моделей.
- Mistral ИИ Studio: бесплатный режим для активации Studio и API-ключа.
- Groq API: free plan с лимитами запросов и токенов.
- Hugging Face Inference Providers: единый API с free tier для разных задач и провайдеров.
- OpenRouter: единый API к множеству моделей, есть модели с суффиксом `:free` и pricing `0`.
- Cohere API: trial/evaluation keys с бесплатными, но ограниченными вызовами.
- Cloudflare Workers ИИ: бесплатная дневная квота Workers AI.
- Ollama: локальный бесплатный API на компьютере/сервере пользователя, без внешнего облачного биллинга.
- LM Studio: локальный OpenAI-compatible API для моделей на своем компьютере.

Для продакшена лучше сделать слой `providers`: он будет хранить API-ключи, лимиты, стоимость, fallback-модели и статус доступности.

## Vercel API слой

Проект подготовлен для Vercel:

- статические страницы остаются в корне проекта;
- серверные функции находятся в папке `api/`;
- первый рабочий маршрут: `POST /api/chat`;
- диагностический маршрут: `GET /api/status`;
- реальные ключи хранятся только в Vercel Environment Variables.

Переменные окружения:

- `GROQ_API_KEY`, `GROQ_MODEL`
- `HF_TOKEN`, `HF_MODEL`
- `OPENROUTER_API_KEY`, `OPENROUTER_MODEL`
- `GIGACHAT_AUTH_KEY`, `GIGACHAT_SCOPE`, `GIGACHAT_MODEL`
- `YANDEX_API_KEY`, `YANDEX_FOLDER_ID`, `YANDEX_MODEL`

`YANDEX_FOLDER_ID` обязателен для YandexGPT. Если его нет, провайдер Yandex считается неготовым и не выбирается автоматически.
